# 🚀 Единая точка запуска

## Для первого запуска (с нуля)

```bash
make bootstrap
```

Эта команда:
1. ⚠️ **Удалит все данные** (попросит подтверждение)
2. ✅ Создаст чистую базу данных
3. ✅ Установит модуль IC UrFU
4. ✅ Загрузит demo subjects (13 дисциплин)
5. ✅ Создаст учетки из `ic_urfu_module/config/demo_credentials.py`
6. ✅ Создаст демо-план для студента

**Результат:** Полностью рабочая система с учетками `student`/`student` и `teacher`/`teacher`

---

## Для обновления существующей установки

```bash
make init
```

Эта команда:
1. ✅ Запустит контейнеры (если не запущены)
2. ✅ Дождется готовности PostgreSQL
3. ✅ Установит модуль (если не установлен)
4. ✅ **Обновит/создаст учетки** из `ic_urfu_module/config/demo_credentials.py`
5. ✅ Синхронизирует пароли

**Результат:** Гарантированно рабочие учетки с паролями из конфига

---

## Сравнение команд

| Команда | Когда использовать | Удаляет данные? | Время |
|---------|-------------------|-----------------|-------|
| `make bootstrap` | Первый запуск или полный сброс | ⚠️ ДА | ~2-3 мин |
| `make init` | Обновление учеток, исправление проблем | ❌ НЕТ | ~30 сек |

## Пошаговое использование

### Вариант 1: Первый раз запускаешь проект

```bash
# 1. Клонировать репозиторий
git clone <repo-url>
cd ic-urfu-odoo-module

# 2. (Опционально) Изменить пароли
vim ic_urfu_module/config/demo_credentials.py

# 3. Запустить bootstrap
make bootstrap
# Введи 'yes' для подтверждения

# 4. Открыть браузер
# http://localhost:8069
# Войти: student / student
```

### Вариант 2: Проект уже установлен, нужно обновить учетки

```bash
# 1. Изменить пароли (если нужно)
vim ic_urfu_module/config/demo_credentials.py

# 2. Запустить init (сохранит данные!)
make init

# 3. Войти с новыми паролями
# http://localhost:8069
```

### Вариант 3: Что-то сломалось, нужно начать с нуля

```bash
# Полный сброс
make bootstrap
# Введи 'yes'
```

## Что происходит внутри

### `make bootstrap`

```
1. docker compose down -v  ← Удаляет контейнеры и volumes
2. make init              ← Запускает полную инициализацию
```

### `make init`

```
1. python3 generate_demo_users.py    ← Генерирует demo_users.xml
2. docker compose up -d                ← Запускает контейнеры
3. Ждет готовности PostgreSQL
4. Ждет готовности Odoo
5. Устанавливает модуль (если нужно)
6. Перезапускает Odoo
7. Загружает данные из demo_credentials.py  ← КЛЮЧЕВОЙ ШАГ
```

Шаг 7 гарантирует, что пользователи создаются/обновляются с правильными паролями из `demo_credentials.py`!

## Troubleshooting

### "Wrong password" после bootstrap/init

```bash
make fix-passwords
```

### Порт 8069 занят

Останови существующий Odoo или измени порт в `local-env/docker-compose.yml`

### Docker не запущен

```bash
# Запусти Docker Desktop
docker info  # Проверь что работает
```

### База не создается

```bash
# Проверь логи
make logs

# Полный сброс
make bootstrap
```

# Сводка доработок: ЗЕТ и модульные спецкурсы

**Дата:** апрель 2026  
**Версия модуля:** `17.0.1.0.7`  
**Язык:** русский.

---

## Задача

- Ровнее валидировать **ЗЕТ** относительно аудиторных часов и глобальных/шаблонных лимитов.  
- Поддержать **модульные спецкурсы** (цепочка частей) с полуавтоматическим добавлением продолжения в следующий семестр.

---

## ЗЕТ и соответствие часам

1. **Настройки** (`ic.urfu.config.settings` и `views/ic_urfu_config_views.xml`):
   - `ic_urfu.hours_per_zet` — сколько ауд. часов соответствует одному ЗЕТ при проверке (по умолчанию **12**, согласовано с демо-справочником дисциплин; для нормативной схемы «36 ч / 1 ЗЕТ» значение задаётся вручную).
   - `ic_urfu.zet_hours_tolerance` — допуск в зачётных единицах между указанным ЗЕТ и расчётом по часам.
   - `ic_urfu.enforce_hours_zet` — включение/выключение проверки соответствия часов и ЗЕТ (по умолчанию **включено**).

2. **Расчёт «ожидаемого» ЗЕТ** (`_expected_zet_from_hours`) — округление `часы / hours_per_zet` с ограничением диапазоном кредитов из `constants`.

3. **Модель дисциплины** (`ic.urfu.subject`): вычисляемое поле **`zet_from_hours`** — подсказка пользователю по текущей норме из настроек.

4. **Валидация в плане** (`IndividualPlan._validate_zet_constraints`):
   - Сначала проверка **час ↔ ЗЕТ** по всем дисциплинам семестров (`_zet_hour_consistency_errors`), если включена в настройках.
   - **Семестр без дисциплин** — ошибка.
   - **Максимум ЗЕТ за семестр** — всегда от глобального параметра `ic_urfu.max_semester_zet`.
   - **Минимум за семестр**: если у шаблона программы для этого номера семестра задан `min_zet > 0`, используется он; иначе — глобальный `ic_urfu.min_semester_zet` (исправлена логика «шаблон vs глобальные лимиты», чтобы не было «дыр» для семестров без строки шаблона).

5. **Генерация DOCX** (`action_generate_document`): перед созданием файла вызывается та же **`_validate_zet_constraints`**, что и при отправке плана — несогласованный план не получит документ без прохождения тех же правил ЗЕТ.

**Затронутые файлы:** `ic_urfu_module/constants.py`, `ic_urfu_module/models/ic_urfu.py`, `ic_urfu_module/models/ic_urfu_config.py`, `ic_urfu_module/views/ic_urfu_config_views.xml`.

## Модульные спецкурсы

1. **Поля в `ic.urfu.subject`:**
   - `is_modular` — признак модульного спецкурса (только для дисциплин **«По выбору»**).
   - `next_module_id` — следующая часть модуля (Many2one на `ic.urfu.subject`, домен по элективам).
   - Ограничения: нельзя сослаться на себя; «следующая часть» только при флаге модульности; тип — электив.

2. **Действие плана** `action_suggest_next_modules` (только статус **черновик**): для каждого семестра ищет дисциплины с заполненной следующей частью и при наличии семестра с номером **+1** добавляет продолжение в **выборные** дисциплины этого семестра без дублей.

3. **Интерфейс:** форма дисциплины и кнопка плана **«Продолжения модулей»** (`views/ic_urfu_views.xml`).

4. **Демо:** пары записей «Продвинутое машинное обучение — модуль 1/2»; в демо-плане в 1-м семестре добавлен модуль 1 — для проверки кнопки во 2-м семестре.

**Затронутые файлы:** `ic_urfu_module/models/ic_urfu.py`, `ic_urfu_module/views/ic_urfu_views.xml`, `ic_urfu_module/demo/demo_subjects.xml`, `ic_urfu_module/demo/demo_plans.xml`.

## Как проверить вручную

1. **Настройки → IC UrFU:** задать норму часов/ЗЕТ, допуск и флаг проверки.  
2. Искусственно сменить у дисциплины в плане ЗЕТ при неизменных часах → **Отправить на проверку** должна давать ошибку (при включённой проверке).  
3. Одобренный план → **Сгенерировать документ** — те же ошибки ЗЕТ, если данные неконсистентны.  
4. Черновик с модулем 1 в семестре *N*, без части 2 в *N+1* → **Продолжения модулей** — часть 2 появляется среди выборных.

## Связь с `CURSOR_2_ITERATION.md`

Функциональность пересекается с **задачами 4 и 5** итерации (валидация ЗЕТ, модульные курсы). Реализация может отличаться в деталях имён сообщений и отсутствии отдельных полей «макс. ЗЕТ в шаблоне семестра» — глобальный максимум за семестр задан через `ir.config_parameter`.

# Деплой на Render.com

Это руководство описывает, как задеплоить Odoo модуль UrFU на Render.com для демонстрации.

## Подготовка

1. Зарегистрируйтесь на [Render.com](https://render.com) (можно через GitHub)
2. Убедитесь, что ваш репозиторий загружен на GitHub

## Автоматический деплой через Blueprint

### Шаг 1: Подключите репозиторий к Render

1. Откройте [Render Dashboard](https://dashboard.render.com)
2. Нажмите **"New +"** → **"Blueprint"**
3. Подключите ваш GitHub репозиторий
4. Render автоматически обнаружит файл `render.yaml` в корне проекта

### Шаг 2: Разверните сервисы

1. Render создаст два сервиса:
   - **odoo-db**: PostgreSQL база данных (Managed, бесплатный tier на 90 дней)
   - **odoo-urfu-demo**: Веб-сервис с Odoo приложением

2. Нажмите **"Apply"** для создания сервисов

3. Дождитесь завершения деплоя (может занять 5-10 минут при первом запуске)

### Шаг 3: Получите URL приложения

После успешного деплоя:
1. Перейдите в сервис **odoo-urfu-demo**
2. URL будет в формате: `https://odoo-urfu-demo.onrender.com`

## Ручной деплой (альтернативный способ)

Если вы не хотите использовать Blueprint, можете развернуть вручную:

### 1. Создайте PostgreSQL базу данных

1. **New +** → **PostgreSQL**
2. Настройки:
   - Name: `odoo-db`
   - Database: `odoo`
   - User: `odoo`
   - Region: Frankfurt (или ближайший к вам)
   - Plan: **Free**
3. Нажмите **Create Database**

### 2. Создайте Web Service

1. **New +** → **Web Service**
2. Подключите GitHub репозиторий
3. Настройки:
   - Name: `odoo-urfu-demo`
   - Region: Frankfurt
   - Branch: `main` (или `doc-generation`)
   - Runtime: **Docker**
   - Plan: **Free**
4. Environment Variables (добавьте вручную):
   ```
   DB_HOST=<копируйте из Internal Database URL>
   DB_PORT=5432
   DB_USER=odoo
   DB_PASSWORD=<копируйте из базы данных>
   DB_NAME=odoo
   ADMIN_PASSWORD=<придумайте сильный пароль>
   ```
5. Нажмите **Create Web Service**

## Первый запуск Odoo

После деплоя:

1. Откройте URL вашего приложения
2. Вы увидите **Database Manager**
3. Создайте новую базу данных:
   - **Master Password**: используйте `ADMIN_PASSWORD` из переменных окружения
   - **Database Name**: `odoo` (или любое имя)
   - **Email**: ваш email
   - **Password**: пароль администратора Odoo
   - **Language**: Russian / Русский
   - **Demo data**: Load demonstration data (опционально)
4. Нажмите **Create database**

## Установка модуля ic_urfu_module

1. После создания базы войдите в Odoo
2. Перейдите в **Apps** (Приложения)
3. Уберите фильтр "Apps" и включите "Extra" в поиске
4. Найдите **ic_urfu_module**
5. Нажмите **Install**

## Важные замечания

### Бесплатный tier Render.com

- **PostgreSQL**: Бесплатно 90 дней, затем $7/месяц
- **Web Service**: Засыпает после 15 минут неактивности, пробуждается при новом запросе (холодный старт ~30 секунд)
- **Ограничения**: 750 часов работы в месяц, 512MB RAM, нет персистентного хранилища (только база данных)

### Персистентность данных

- ⚠️ Render бесплатный tier НЕ сохраняет файлы между деплоями
- Данные в PostgreSQL сохраняются
- Загруженные файлы и вложения НЕ сохранятся
- Для production используйте S3 или платный план

### Производительность

- Холодный старт может занять 30-60 секунд
- После прогрева приложение работает быстро
- Для постоянной работы без засыпания нужен платный план ($7/месяц)

## Мониторинг и логи

1. Откройте Render Dashboard
2. Перейдите в ваш Web Service
3. Вкладка **Logs** покажет логи Odoo в реальном времени
4. Вкладка **Events** покажет историю деплоев

## Обновление приложения

Render автоматически пересобирает приложение при каждом push в GitHub:

1. Сделайте изменения в коде
2. Закоммитьте и запушьте в GitHub
3. Render автоматически запустит новый деплой
4. Следите за процессом в разделе **Events**

## Удаление деплоя

1. Откройте Render Dashboard
2. Перейдите в сервис
3. Settings → **Delete Service**
4. Подтвердите удаление

## Поддержка

Если возникнут проблемы:
- Проверьте логи в Render Dashboard
- Убедитесь, что все переменные окружения заданы правильно
- Проверьте, что база данных доступна
- Обратитесь к [документации Render](https://render.com/docs)
